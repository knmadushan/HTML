    2. Answer
    • DBMS සැලසුම එහි Architecture මත රඳා පවතී. මූලික Client/Server Architecture පරිගණක, වෙබ් සේවාදායකයන්, දත්ත සමුදා සේවාදායකයන් සහ ජාල සමඟ සම්බන්ධ වී ඇති අනෙකුත් සංරචක විශාල ප්‍රමාණයක් සමඟ කටයුතු කිරීමට භාවිතා කරයි.

  3. Architecture වර්ග විස්තර කරන්න
    4. Answer
    • 1-tier architecture:මෙම Architecture තුළ, දත්ත සමුදාය පරිශීලකයාට සෘජුවම ලබා ගත හැකිය. එයින් අදහස් වන්නේ පරිශීලකයාට කෙලින්ම DBMS මත සිට එය භාවිතා කළ හැකි බවයි.


    • 2-tier architecture:2-tier architecture මූලික client-server ට සමාන වේ. 2-tier architecture තුළ, සේවාදායක කෙළවරේ ඇති යෙදුම් සේවාදායක පැත්තේ ඇති දත්ත සමුදාය සමඟ සෘජුවම සන්නිවේදනය කළ හැකිය. මෙම අන්තර්ක්‍රියාව සඳහා, ODBC, සහ JDBC වැනි API භාවිතා වේ.


    • 3-tier architecture:3-tier architecture සේවාදායකයා (Client) සහ සේවාදායකය (Server) අතර තවත් ස්ථරයක් අඩංගු වේ. මෙම Architecture තුළ, සේවාදායකයාට සේවාදායකය සමඟ සෘජුව සන්නිවේදනය කළ නොහැක.
